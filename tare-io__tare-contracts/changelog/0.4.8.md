# 0.4.8 Release

Changes since version 0.4.4.

## Breaking Changes

### Timestamps narrowed: `uint64` → `uint48`

All `timestamp` parameters and the `Entry.timestamp` field changed from `uint64` to `uint48`.

Affected functions: `create`, `fund`, `accrue`, `receiveBorrowerPayment`, `chargeMiscFee`, `applyWaterfall`, `investorWithdraw`, `servicerWithdraw`, `createInternalEntry`, `updateLoanData`.

### `create()` and `updateLoanData()` require a `timestamp` parameter

Previously used `block.timestamp` internally. Callers must now pass `uint48 timestamp` as the last argument.

### Investor ownership moved to ERC-721 NFT

- The `investors` storage mapping was removed from `Loans.sol`. A compatibility view function `investors(uint64 loanId)` remains but reads from `loansNFT.ownerOf(loanId)`.
- New contract **`LoansNFT.sol`** (ERC721Enumerable). Minted automatically during `create()`. The NFT token ID equals the loan ID.
- Admin must call `setLoansNFT(address)` once after deployment (cannot be changed once set).

### Withdrawal functions restructured

| Old                                                    | New                                  | Notes                                                      |
| ------------------------------------------------------ | ------------------------------------ | ---------------------------------------------------------- |
| `investorWithdraw(loanId, principal, interest, ...)`   | **Removed**                          | Single-loan version deleted                                |
| `investorWithdrawBatch(loanIds[], ...)`                | `investorWithdraw(loanIds[], ...)`   | Renamed; now the only investor withdrawal function         |
| `servicerWithdraw(loanId, miscFee, servicingFee, ...)` | `servicerWithdraw(loanIds[], ...)`   | Now batch-only; auto-withdraws all available fees per loan |
| —                                                      | `originatorWithdraw(loanIds[], ...)` | **New** — batch originator fee withdrawal                  |
| `withdrawToRole(loanId, role, ...)`                    | **Removed**                          | Generic role withdrawal deleted                            |

All withdrawal functions return structured result arrays: `InvestorWithdrawalResult[]`, `ServicerWithdrawalResult[]` (new struct), or `OriginatorWithdrawalResult[]` (new struct).

### `ENTRY_ORIGINATION_FEE` renamed

Renamed to `ENTRY_ORIGINATOR_FEE_WITHHOLDING` (value remains `6`).

## New Contracts

### `LoansNFT.sol`

ERC-721 Enumerable contract representing loan investor positions. Token ID = loan ID. Only the `Loans` contract can mint. Admin (checked via `Loans.isAdmin`) can update the base URI.

### `LoansExchange.sol`

Peer-to-peer loan sale marketplace using the NFT:

- `createOffer(buyer, price, deadline, loanIds[])` — seller escrows NFTs into the contract, creates a directed offer.
- `acceptOffer(offerId)` — buyer pays USDC, receives NFTs.
- `cancelOffer(offerId)` — seller reclaims NFTs.
- Buyer must be pre-registered as an investor via `ILoansAuth.isRegisteredForRole`.

## New Functions

### `returnFunds()` (Loans.sol)

Allows the servicer to deposit funds back into a loan's cash account, crediting one of: `ACC_SERVICER_FEE_PAID`, `ACC_SERVICER_MISC_FEE_PAID`, or `ACC_SERVICER_ERROR_CORRECTION`.

### `executeTrustedCallBatch()` (TrustedCalls.sol)

Batch execution of multiple trusted calls in a single transaction.

## New Constants

| Constant                          | Value | Type                |
| --------------------------------- | ----- | ------------------- |
| `ACC_SERVICER_ERROR_CORRECTION`   | `208` | Account (Liability) |
| `ENTRY_ORIGINATOR_FEE_WITHDRAWAL` | `17`  | Entry type          |
| `ENTRY_SERVICER_FUND_RETURN`      | `18`  | Entry type          |

## New Structs

- `OriginatorWithdrawalResult { uint64 loanId; int128 amount; }`
- `ServicerWithdrawalResult { uint64 loanId; int128 miscFee; int128 servicingFee; }`

## New Errors

- `AlreadyInitialized()` — thrown if `setLoansNFT` is called after the NFT contract is already set.

## Internal Changes

- `SmartAccountFactory` refactored to use delegatecall-based configuration (external `deploy()` signature unchanged).
- Date validation added in `_updateLoanData`: `maturityDate > originationDate` and `originationDate < maturityDate` are now enforced.
- `withLoanUpdate` modifier replaces manual `updatedAt` assignments across functions.
